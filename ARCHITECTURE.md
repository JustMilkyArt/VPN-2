# VPN Архитектура — Справочный документ

## Типы подключений

### Прямые (3 штуки)

```
1. VLESS Reality (прямой)
   Пользователь ──[VLESS Reality]──► EU сервер ──► интернет

2. AmneziaWG (прямой)
   Пользователь ──[AWG]──► EU сервер ──► интернет

3. NaiveProxy (прямой)
   Пользователь ──[NaiveProxy]──► EU сервер ──► интернет
```

### Каскадные (3 штуки)

```
4. VLESS → каскад
   Пользователь ──[VLESS]──► RU сервер ──[VLESS]──► EU сервер ──► зарубежный интернет
                                  └──────────────────────────────► российский интернет
                                  └──[WARP фолбэк если EU упал]──► Cloudflare ──► интернет

5. AWG → каскад
   Пользователь ──[AWG]──► RU сервер ──[VLESS]──► EU сервер ──► зарубежный интернет
                                └──────────────────────────────► российский интернет
                                └──[WARP фолбэк если EU упал]──► Cloudflare ──► интернет

6. NaiveProxy → каскад
   Пользователь ──[NaiveProxy]──► RU сервер ──[VLESS]──► EU сервер ──► зарубежный интернет
                                        └──────────────────────────────► российский интернет
                                        └──[WARP фолбэк если EU упал]──► Cloudflare ──► интернет
```

**Ключевая логика каскада:** RU сервер роутит трафик — российские IP выходят напрямую с RU, зарубежные идут через EU.

## Стек по серверам

### EU сервер

| Сервис | Для чего |
|---|---|
| Xray-core | VLESS Reality (прямой №1) + приём каскадов от RU (№4,5,6) |
| AmneziaWG + awg-tools | Прямое подключение №2 |
| NaiveProxy (бинарник Go) | Прямое подключение №3 |
| Caddy | Фронтенд для NaiveProxy (авто SSL) |
| SSL-сертификат | Только для NaiveProxy |
| Домен (A-запись) | Только для NaiveProxy |
| WARP | ❌ НЕ нужен |

### RU сервер

| Сервис | Для чего |
|---|---|
| Xray-core | Приём пользователя (каскад №4) + исходящий туннель на EU (для №4,5,6) |
| AmneziaWG + awg-tools | Приём пользователя (каскад №5) |
| NaiveProxy (бинарник Go) | Приём пользователя (каскад №6) |
| Caddy | Фронтенд для NaiveProxy (авто SSL) |
| SSL-сертификат | Только для NaiveProxy |
| Домен (A-запись) | Только для NaiveProxy |
| WARP (warp-cli) | Фолбэк если EU недоступен |
| Роутинг RU/EU | Решает куда направить трафик |

## Оптимальные параметры протоколов

### VLESS Reality

| Параметр | Оптимальное значение |
|---|---|
| Транспорт | TCP |
| TLS | Reality |
| serverName | Домен географически близкий к серверу (EU в Финляндии → финский/европейский CDN) |
| publicKey / privateKey | Генерируются через `xray x25519` |
| shortId | Случайный hex |
| Порт | 443 |
| Flow | `xtls-rprx-vision` |
| fingerprint | Рандомный из: chrome, firefox, safari, edge |
| Fallback | Xray возвращает HTML-заглушку при прямом обращении браузером |

### AmneziaWG

| Параметр | Оптимальное значение |
|---|---|
| Транспорт | UDP |
| Порт | 51820 |
| PrivateKey / PublicKey | `awg genkey` / `awg pubkey` |
| PresharedKey | `awg genpsk` |
| Jc | 4 |
| Jmin | 40 |
| Jmax | 70 |
| S1 | Случайное 15–150 |
| S2 | Случайное 15–150 |
| H1-H4 | Случайные уникальные числа |
| MTU | 1420 |
| PersistentKeepalive | 25 сек |
| AllowedIPs (прямой) | `0.0.0.0/0, ::/0` |
| AllowedIPs (каскад) | Избирательно по маршруту |
| DNS | `1.1.1.1, 8.8.8.8` |

### NaiveProxy

| Параметр | Оптимальное значение |
|---|---|
| Транспорт | HTTP/2 (h2) поверх TLS |
| Порт | 443 |
| TLS | Реальный Let's Encrypt (через Caddy автоматически) |
| Аутентификация | Basic Auth (username:password) |
| Фронтенд | Caddy (не nginx — авто SSL и лучшая маскировка) |
| Маскировка | Выглядит как обычный HTTPS-сайт |

### WARP (только RU, фолбэк)

| Параметр | Оптимальное значение |
|---|---|
| Клиент | warp-cli (официальный, не cloudflared) |
| Режим | WARP routing mode (не proxy-mode) |
| Порт | 2408 (стандартный Cloudflare) |
| Ключи | Выдаются автоматически Cloudflare |
| Лицензия | Бесплатный WARP или WARP+ |

## Домен — где нужен

| Протокол | Домен нужен? |
|---|---|
| VLESS Reality | ❌ Нет (использует чужой TLS-отпечаток) |
| AmneziaWG | ❌ Нет (UDP по IP) |
| NaiveProxy | ✅ Обязателен (реальный HTTPS) |
| WARP | ❌ Нет (Cloudflare управляет) |
